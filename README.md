# Установка Remnawave Node одной командой

**Русский** · [English](README_EN.md)

Скрипт автоматически ставит **Docker**, **nano** и зависимости, затем открывает редактор для вставки готового **`docker-compose.yml`** из панели [Remnawave](https://docs.rw/docs/install/remnawave-node/). После сохранения файла контейнер ноды запускается сам — руками собирать compose на сервере не нужно.

**ОС: рассчитано на Ubuntu** (типичный VPS: Ubuntu 22.04 / 24.04 LTS). Та же логика подходит **Debian** и другим дистрибутивам с **`apt-get`**; на RHEL/Alpine/Arch без `apt` пакеты `nano`/`curl` и Docker нужно поставить вручную, затем запустить скрипт.

Подходит любому, кто разворачивает ноду на **новом VPS** — шаги одинаковые.

---

## Что понадобится

| Требование | Зачем |
|------------|--------|
| Сервер (VPS) с **Linux** | Где крутится нода |
| Доступ **root** по **SSH** | Скрипт ставит пакеты и Docker |
| **Ubuntu** (рекомендуется) или **Debian** с `apt` | Автоустановка `nano`, `curl` через `apt-get` |
| Обычная SSH-сессия в терминале | Нужен **интерактивный** режим (откроется **nano**) |
| Уже созданная нода в панели Remnawave | Оттуда копируется YAML |

---

## Шаг 1 — Панель Remnawave (сделай до захода на сервер)

1. Войди в **Remnawave Panel**.
2. Открой **Nodes** → **Management**.
3. Нажми **+** (создать ноду), заполни поля (порт ноды, имя и т.д.).
4. Нажми **Copy docker-compose.yml** — в буфер обмена попадёт готовый файл.
5. **Пока не нажимай «Create» до конца мастера** — сначала установи ноду на сервере (см. шаг 2), затем вернись в панель и завершишь создание (шаг 3).

Сохрани скопированный текст куда-нибудь временно или оставь в буфере — он понадобится в nano.

---

## Шаг 2 — Сервер: одна команда установки

Подключись к VPS по SSH (PuTTY, Terminal, Windows Terminal — не важно, главное **полноценный терминал**).

Выполни **три строки подряд** (можно скопировать целиком):

```bash
curl -fsSL https://raw.githubusercontent.com/Shivarin/remnanode-installer/main/install.sh -o /root/install-remnanode.sh
chmod +x /root/install-remnanode.sh
sudo /root/install-remnanode.sh
```

**Почему не `curl ... | bash`:** если вывод `curl` сразу передать в **`bash`**, скрипт выполняется **без интерактивного терминала** — **nano не откроется**. Сначала сохрани файл (`-o /root/install-remnanode.sh`), затем запусти **`bash /root/install-remnanode.sh`** — так и задумано.

### Что произойдёт дальше (автоматически)

1. Установятся **nano**, **curl**, **ca-certificates** (на **Ubuntu / Debian** через `apt`).
2. Если Docker ещё нет — он установится скриптом с [get.docker.com](https://get.docker.com).
3. Создаётся каталог **`/opt/remnanode`**, файл **`docker-compose.yml`**.
4. Откроется **nano**: **вставь сюда целиком** то, что скопировал из панели (**Ctrl+Shift+V** / правая кнопка в зависимости от клиента).
5. Сохрани файл: **Ctrl+O**, **Enter**, выйди: **Ctrl+X**.
6. Запустятся **`docker compose pull`** и **`docker compose up -d`** — нода поднимется в Docker.

Если файл оставить пустым и выйти — скрипт остановится с ошибкой: нужно вставить YAML из панели.

---

## Шаг 3 — Снова панель: завершить создание ноды

1. Вернись в карточку этой ноды в Remnawave.
2. Нажми **Next**, выбери **Config Profile**.
3. Нажми **Create** (или аналог по версии панели).

Без этого панель может не связать профиль конфигурации с нодой.

---

## Фаервол

На сервере ноды открой порт **`NODE_PORT`** (тот, что указан в `docker-compose` / в панели) **только для IP сервера с панелью** Remnawave — иначе панель не достучится до API ноды. Подробности: [официальная документация](https://docs.rw/docs/install/remnawave-node/).

## Проверка и логи

```bash
cd /opt/remnanode && docker compose ps
cd /opt/remnanode && docker compose logs -f -t
```

---

## Если что-то пошло не так

| Проблема | Что сделать |
|----------|-------------|
| Скрипт сразу пишет про **нет TTY** | Не используй `curl ... | bash` без нормального SSH-терминала; зайди по SSH и выполни три команды из шага 2. |
| **nano** не открылся | Убедись, что сессия SSH интерактивная (не некоторые автоматизации без псевдотерминала). |
| Не **Ubuntu / Debian** (нет `apt`) | Установи Docker и `curl` вручную по документации дистрибутива, затем снова `sudo bash install.sh`. |
| Ошибка **docker compose** | Проверь, что YAML из панели вставлен полностью, без обрезки. |

## Альтернатива: клонировать репозиторий

```bash
git clone https://github.com/Shivarin/remnanode-installer.git
cd remnanode-installer
sudo bash install.sh
```

Дальше тот же сценарий с **nano** и вставкой YAML.

## Дополнительные переменные (редко)

| Переменная | По умолчанию |
|------------|----------------|
| `INSTALL_DIR` | `/opt/remnanode` |
| `COMPOSE_FILE` | `docker-compose.yml` |

Пример другого каталога:

```bash
sudo env INSTALL_DIR=/srv/remnanode bash /root/install-remnanode.sh
```

(путь к скрипту укажи свой, если отличался.)

## Безопасность

Не выкладывай в git и не публикуй **`docker-compose.yml`** с сервера, если там есть **секреты** (ключи, токены) — это полный доступ к ноде.
